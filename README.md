# Passage

Discover destinations calibrated to your passport.

A demo Next.js 16 app — onboarding flow, AMP (AI Mobility Profile) scoring,
swipe-deck destination discovery, itinerary editor, and a fake booking flow
backed by a route handler.

## Stack

- Next.js 16 (App Router, Turbopack)
- React 19
- Tailwind CSS 3
- Zustand (persisted to `localStorage`)
- Framer Motion
- Serwist (PWA / offline caching)

## Scripts

```bash
npm run dev          # local dev server (http://localhost:3000)
npm run dev:host     # bind to 0.0.0.0 for testing on a phone on the LAN
npm run build        # production build
npm run start        # serve the production build
npm run lint         # ESLint (flat config)
npm run lint:fix     # ESLint with --fix
npm run typecheck    # tsc --noEmit
npm run format       # Prettier write
npm run format:check # Prettier check
```

## Layout

```
app/                 routes (App Router)
  api/book/          fake booking endpoint
  trip/[id]/         destination detail (async params)
  sw.ts              Serwist service worker source
components/          UI components
lib/
  amp.ts             AMP scoring + tier metadata
  data.ts            countries, profile slides, destination dataset
  store.ts           Zustand store with localStorage persistence
  types.ts           shared types
```

## Notes

- `sw.js` in `public/` is generated by Serwist at build time and is gitignored.
- The store hydrates from `localStorage`; pages render `null` until
  `_hasHydrated` flips, to avoid SSR / client mismatches.
- Theme switching is driven by a `data-theme` attribute on `<html>` and
  CSS variables in `app/globals.css`. Tailwind tokens (`bg-bg`, `text-fg`,
  `border-ghost`, …) resolve through these variables.
